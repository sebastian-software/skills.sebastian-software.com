# I18n UX

Design global interfaces so language, country, currency, units, timezone, direction, and shipping/payment context can differ. Inferred locale is a starting point, not a command.

## Working Rules

- Separate language choice from country, region, currency, units, timezone, and fulfilment context.
- Make locale controls discoverable, reversible, keyboard-accessible, and screen-reader-friendly.
- Support translated text expansion, RTL, localized punctuation, localized quotes, and changed reading order.
- Test forms, navigation, tables, and component labels with long translated strings.

## Locale Decision Rules

- Infer locale as a starting suggestion, not as a forced redirect.
- Keep language, country, currency, units, timezone, tax, fulfilment, and legal region as separate concepts unless the product domain explicitly ties them together.
- Let users change locale preferences after automatic detection.
- Preserve the current task when changing language or region.
- Use BCP 47 language tags and platform/Intl APIs for formatting dates, numbers, currencies, lists, and relative time.
- Avoid concatenating translated fragments. Design messages with placeholders that translators can reorder.

## UI Stress Tests

Before calling a global UI ready:

- Test German-length labels and compound nouns.
- Test an RTL locale with icons, arrows, breadcrumbs, tables, forms, and side navigation.
- Test mixed-direction content such as names, codes, phone numbers, and addresses.
- Test currency, date, time, timezone, measurement, and pluralization edge cases.
- Test locale selectors with keyboard and screen readers.
- Ensure truncation does not remove the only distinguishing part of a label or option.

## Component Requirements

- Components must survive text expansion without clipped labels, overlapping controls, or hidden primary actions.
- Icons that imply direction must mirror only when the meaning is directional, not when they represent an object or brand.
- Tables and data grids need localized alignment, number formatting, and headers that still identify cells after responsive transformation.
- Forms need localized input examples, address structures, names, phone numbers, postal codes, and validation copy.

## Additional Rules

- Language/country/currency/units preferences, avoid forced redirects, decouple inferred presets, allow overrides, make selectors discoverable and accessible, and define component requirements for global preference selectors.
- Narrow durable guidance for language-dependent quotes and CLDR-driven punctuation via CSS, useful as localized content-polish note rather than core design-system rule.
