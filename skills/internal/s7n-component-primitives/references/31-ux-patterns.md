# UX Patterns

Choose interaction patterns from the user's task, navigation model, content volume, recovery path, and accessibility requirements. Familiar patterns are useful only when their tradeoffs match the workflow.

## Working Rules

- Choose pagination, load more, infinite scroll, filtering, and search from the user's need to compare, resume, scan, or complete.
- Preserve URL state, scroll position, focus position, and back-button behavior when lists or navigation change.
- Keep menus, navigation, footers, and recovery actions reachable.
- Make pattern behavior explicit for keyboard, screen-reader, touch, and reduced-motion contexts.

## Additional Rules

- Infinite scroll vs pagination/load-more, position and URL preservation, footer reachability, focus/keyboard/screenreader handling, announcements, performance, and list/feed component state.
- Navigation/menu design as information architecture and orientation, keep navigation visible on larger screens, use expected locations, show current location, provide local navigation for related content, use clear scanable labels, ensure contrast and target size, signify submenus, prefer click/tap over hover-only behavior, avoid deep cascading menus and gimmicky navigation; connect to component-development, responsive-design, html-accessibility, and design-system.
- Priority-plus navigation affects responsive navigation and disclosure.
- Use elastic overflow scrolling as interaction/platform behavior context.
- Use dashboard design for product UI density, decluttering, research, and data-viz tradeoffs.
