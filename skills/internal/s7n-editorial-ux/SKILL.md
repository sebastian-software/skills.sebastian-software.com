# S7N Editorial UX

Use this skill for words inside product UI: labels, helper text, empty states, headings, button text, status messages, and short-form explanation.

## Workflow

1. Identify the user's question at that moment: what is this, why does it matter, what happens next?
2. Remove copy that explains the interface instead of making the interface clearer.
3. Make labels concrete and action text outcome-oriented.
4. Match tone to consequence: calm for errors, direct for actions, sparse for repeated workflows.
5. Check text expansion, localization, and accessibility names.

## Rules

- Button labels should usually be verb plus object.
- Error copy should name the problem and the recovery path.
- Empty states should help users take the next useful action.
- Avoid marketing language inside task surfaces.
- Prefer specific nouns over clever phrasing.
