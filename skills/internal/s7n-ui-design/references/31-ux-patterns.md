# UX Patterns

Choose interaction patterns from the user's task, navigation model, content volume, recovery path, and accessibility requirements. Familiar patterns are useful only when their tradeoffs match the workflow.

## Working Rules

- Choose pagination, load more, infinite scroll, filtering, and search from the user's need to compare, resume, scan, or complete.
- Preserve URL state, scroll position, focus position, and back-button behavior when lists or navigation change.
- Keep menus, navigation, footers, and recovery actions reachable.
- Make pattern behavior explicit for keyboard, screen-reader, touch, and reduced-motion contexts.
