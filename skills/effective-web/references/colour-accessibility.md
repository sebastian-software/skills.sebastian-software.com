# Accessible Color Decisions

Use this module when color carries hierarchy, action, status, or feedback. A
palette is correct only when its semantic states remain understandable.

## Establish Meaning Beyond Hue

- Never use color alone for a status, validation result, required field, or
  interactive affordance. Pair it with text, icon shape, placement, or pattern.
- Verify contrast for normal and large text, icons that carry meaning, borders
  that establish state, focus indicators, and interaction states—not only a
  static brand swatch.
- Use semantic roles such as text, surface, border, action, focus, success,
  warning, and danger. A brand hue can implement a role, but it is not the role.

## Test the Whole State Set

- Check light mode, dark mode, forced colors, disabled controls, hover/focus,
  selected states, grayscale print, and color-vision differences relevant to the
  product's tasks.
- Preserve contrast when a color is composited over an image, translucent layer,
  or elevation surface. Estimate only to start; use rendered evidence to finish.

For WCAG/APCA context and advanced palette edge cases, consult the [deep color
appendix](colour.md).
