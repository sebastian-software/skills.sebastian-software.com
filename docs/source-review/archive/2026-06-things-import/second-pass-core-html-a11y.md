## Decision Scale

| Decision | Meaning |
| --- | --- |
| `primary` | Can shape rules, checklists, or reference structure directly. |
| `secondary` | Use as supporting example, caveat, or explanatory context. |
| `ignored` | Do not use for skill/reference updates. |
| `needs_discussion` | Needs more context before deciding. |

## Review Notes

- User preference established: explanatory expert articles can be `primary`
  when they explain the "why" and application better than specs or MDN-style
  references. Specs remain valuable for accuracy/currentness, but are not
  automatically better teaching sources.
